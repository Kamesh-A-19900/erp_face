# Design Document: Face Attendance Arduino Display

## Overview

This design converts the existing GUI-based face recognition attendance system into a fully headless pipeline. The Raspberry Pi runs two Python scripts — `recognizer.py` (continuous recognition loop) and `enroller.py` (one-shot enrollment) — that replace `attendance_taker.py` and `get_faces_from_camera_tkinter.py` respectively. All SQLite, tkinter, and cv2.imshow code is removed. Recognized roll numbers are sent over USB serial to an Arduino Mega, which renders them on a 2.4" ILI9341 TFT LCD shield.

The existing dlib ResNet recognition logic and `features_extraction_to_csv.py` are kept unchanged.

### Data Flow

```
Webcam frame
    │
    ▼
dlib frontal face detector
    │ face rectangles
    ▼
dlib shape predictor (68 landmarks)
    │ shape
    ▼
dlib ResNet face_recognition_model_v1
    │ 128-D descriptor
    ▼
Euclidean distance vs. all rows in features_all.csv
    │ min_distance, roll_number
    ▼
Threshold check (< 0.4)
    │ matched roll_number  OR  "unknown"
    ▼
CooldownTracker.should_send(roll_number)
    │ True / False
    ▼ (True only)
SerialSender.send(roll_number)  →  Arduino Mega  →  TFT display
    │
    ▼
logging (INFO: recognition event, DEBUG: serial tx, DEBUG: unknown)
```

## Components and Interfaces

### 1. `config.ini`

```ini
[camera]
device_index = 0

[serial]
port = /dev/ttyUSB0
baud_rate = 9600

[recognition]
confidence_threshold = 0.4

[enrollment]
num_images = 10
```

### 2. `recognizer.py`

#### `load_config(path: str = "config.ini") -> dict`

Reads `config.ini` using `configparser`. Returns a dict with keys:
- `camera_index` (int, default 0)
- `serial_port` (str, default `/dev/ttyUSB0`)
- `baud_rate` (int, default 9600)
- `confidence_threshold` (float, default 0.4)

If the file is absent, logs a WARNING and returns defaults.

#### `load_face_database(csv_path: str) -> tuple[list[str], list[np.ndarray]]`

Reads `data/features_all.csv` using pandas (matching existing logic). Returns:
- `names`: list of roll number strings
- `descriptors`: list of 128-D numpy float arrays

Raises `FileNotFoundError` if the CSV does not exist (caller handles exit).

#### `class CooldownTracker`

Tracks per-roll-number cooldown state.

```python
class CooldownTracker:
    COOLDOWN_SECONDS: int = 30  # hardcoded, not configurable

    def __init__(self) -> None:
        self._last_sent: dict[str, float] = {}

    def should_send(self, roll_number: str, now: float | None = None) -> bool:
        """Return True if roll_number has not been sent within COOLDOWN_SECONDS."""

    def record_sent(self, roll_number: str, now: float | None = None) -> None:
        """Record that roll_number was just sent at time `now` (default: time.time())."""
```

The `now` parameter is injectable for testing.

#### `class SerialSender`

Wraps `serial.Serial` for sending roll numbers.

```python
class SerialSender:
    def __init__(self, port: str, baud_rate: int) -> None:
        """Open serial port. Raises serial.SerialException on failure."""

    def send(self, roll_number: str) -> None:
        """Write roll_number + '\n' encoded as UTF-8 bytes."""

    def close(self) -> None:
        """Close the serial port."""
```

#### `find_best_match(descriptor: np.ndarray, names: list[str], descriptors: list[np.ndarray], threshold: float) -> tuple[str, float]`

```python
def find_best_match(
    descriptor: np.ndarray,
    names: list[str],
    descriptors: list[np.ndarray],
    threshold: float,
) -> tuple[str, float]:
    """
    Compare descriptor against all known descriptors.
    Returns (roll_number, min_distance) if min_distance < threshold,
    else ("unknown", min_distance).
    """
```

#### `euclidean_distance(a: np.ndarray, b: np.ndarray) -> float`

```python
def euclidean_distance(a: np.ndarray, b: np.ndarray) -> float:
    """Compute L2 distance between two 128-D vectors."""
    return float(np.sqrt(np.sum(np.square(a - b))))
```

#### `class Recognizer`

Main orchestration class.

```python
class Recognizer:
    def __init__(self, config: dict) -> None: ...

    def run(self) -> None:
        """
        Main loop:
        1. Load face database (exit 1 on failure)
        2. Open serial port (exit 1 on failure)
        3. Open webcam (exit 1 on failure)
        4. Loop: read frame → detect → recognize → cooldown check → send
        5. On KeyboardInterrupt: close serial, release camera, exit 0
        """
```

#### `main()` entry point

```python
def main() -> None:
    logging.basicConfig(
        level=logging.DEBUG,
        format="%(asctime)s %(levelname)s %(message)s",
    )
    config = load_config()
    recognizer = Recognizer(config)
    recognizer.run()
```

### 3. `enroller.py`

#### `load_enroller_config(path: str = "config.ini") -> dict`

Returns:
- `camera_index` (int, default 0)
- `num_images` (int, default 10)

#### `get_next_person_index(base_dir: str) -> int`

Scans `data/data_faces_from_camera/` for existing `person_<N>_*` directories and returns `max(N) + 1` (or 1 if empty).

#### `class Enroller`

```python
class Enroller:
    def __init__(self, roll_number: str, config: dict) -> None: ...

    def run(self) -> None:
        """
        1. Determine next person index N
        2. Create directory data/data_faces_from_camera/person_<N>_<roll_number>/
        3. Open webcam
        4. Loop until num_images saved:
           - Read frame
           - Detect faces
           - If 0 faces: skip
           - If >1 faces: log warning, skip
           - If exactly 1 face: save cropped image, increment counter
        5. Log completion, exit 0
        """
```

#### `main()` entry point

```python
def main() -> None:
    parser = argparse.ArgumentParser()
    parser.add_argument("--roll", required=True, help="Roll number to enroll")
    args = parser.parse_args()
    config = load_enroller_config()
    enroller = Enroller(roll_number=args.roll, config=config)
    enroller.run()
```

### 4. `arduino/attendance_display/attendance_display.ino`

#### Pin Definitions

```cpp
#define TFT_CS   10
#define TFT_DC    9
#define TFT_RST   8
// MOSI=51, MISO=50, SCK=52 — hardware SPI on Mega, no defines needed
```

#### Libraries

```cpp
#include <Adafruit_GFX.h>
#include <Adafruit_ILI9341.h>
```

#### Global State

```cpp
Adafruit_ILI9341 tft = Adafruit_ILI9341(TFT_CS, TFT_DC, TFT_RST);
String currentRoll = "";
```

#### `setup()`

- Initialize `Serial` at 9600 baud
- Initialize `tft`
- Fill screen black
- Display "Ready" centered in font size 3

#### `loop()`

- Poll `Serial.available()`
- When a `\n`-terminated line arrives, read it via `Serial.readStringUntil('\n')`
- Trim whitespace
- Truncate to 20 characters if longer
- Call `displayRoll(roll)`

#### `displayRoll(String roll)`

- Fill screen black
- Set text size 3, text color white
- Compute centered x/y position:
  - Each char at size 3 is 18px wide, 24px tall
  - `x = (240 - roll.length() * 18) / 2`
  - `y = (320 - 24) / 2`
- `tft.setCursor(x, y)`
- `tft.print(roll)`

#### `centerText(String text, int textSize)`

Helper that computes and sets cursor for centered rendering.

## Data Models

### Face Database Row (CSV)

```
<roll_number>, <f1>, <f2>, ..., <f128>
```

- Column 0: roll number string (e.g., `"21CS001"`)
- Columns 1–128: float values of the 128-D dlib descriptor
- Empty cells treated as `0` (matching existing logic)

### Config Schema

| Section | Key | Type | Default |
|---|---|---|---|
| camera | device_index | int | 0 |
| serial | port | str | /dev/ttyUSB0 |
| serial | baud_rate | int | 9600 |
| recognition | confidence_threshold | float | 0.4 |
| enrollment | num_images | int | 10 |

### Cooldown State

```python
_last_sent: dict[str, float]  # roll_number -> unix timestamp of last send
```

### Serial Message Format

```
<roll_number>\n
```

Encoded as UTF-8. Maximum meaningful length: 20 characters (Arduino truncates beyond that).

## Correctness Properties

*A property is a characteristic or behavior that should hold true across all valid executions of a system — essentially, a formal statement about what the system should do. Properties serve as the bridge between human-readable specifications and machine-verifiable correctness guarantees.*

### Property 1: Face matching correctness

*For any* 128-D descriptor vector and any non-empty face database, `find_best_match` returns the roll number whose descriptor has the minimum Euclidean distance to the query descriptor when that distance is below the threshold, and returns `"unknown"` when all distances are at or above the threshold.

**Validates: Requirements 1.7, 1.8**

### Property 2: Serial message format

*For any* roll number string (non-empty, up to 20 characters), the bytes written to the serial port by `SerialSender.send` are exactly `(roll_number + "\n").encode("utf-8")`.

**Validates: Requirements 2.1**

### Property 3: Cooldown enforcement

*For any* roll number, if `record_sent` is called at time T, then `should_send` called at any time T' where `T' - T < 30` returns `False`. Furthermore, for any two distinct roll numbers A and B, calling `record_sent(A)` does not affect the result of `should_send(B)`.

**Validates: Requirements 2.5, 2.6**

### Property 4: Arduino string truncation

*For any* string of length greater than 20 characters, the truncation logic produces a string of exactly 20 characters that equals the first 20 characters of the original string.

**Validates: Requirements 3.7**

### Property 5: Config round-trip

*For any* valid combination of config values (camera index, serial port, baud rate, confidence threshold), writing those values to a `config.ini` file and reading them back via `load_config` produces a dict with values equal to the originals.

**Validates: Requirements 5.2**

### Property 6: Recognition log content

*For any* recognition event (matched roll number, Euclidean distance, timestamp), the log record emitted at INFO level contains the roll number string, a numeric distance value, and a timestamp.

**Validates: Requirements 6.1**

### Property 7: Enrollment path format

*For any* roll number string and person index N ≥ 1, the enrollment save directory path is exactly `data/data_faces_from_camera/person_<N>_<roll_number>/`.

**Validates: Requirements 4.3**

## Error Handling

| Condition | Component | Behavior |
|---|---|---|
| `features_all.csv` missing | `recognizer.py` startup | Log ERROR, `sys.exit(1)` |
| Webcam cannot be opened | `recognizer.py` startup | Log ERROR, `sys.exit(1)` |
| Serial port cannot be opened | `recognizer.py` startup | Log ERROR, `sys.exit(1)` |
| Serial write failure at runtime | `SerialSender.send` | Log ERROR, re-raise; caller exits |
| Webcam read failure at runtime | recognition loop | Log ERROR, break loop, exit cleanly |
| `config.ini` missing | `load_config` | Log WARNING, return hardcoded defaults |
| No face in enrollment frame | `enroller.py` | Skip frame silently, continue |
| Multiple faces in enrollment frame | `enroller.py` | Log WARNING, skip frame, continue |
| `--roll` argument missing | `enroller.py` | argparse prints usage, exits 2 |

## Testing Strategy

### Dual Testing Approach

Both unit tests and property-based tests are used. Unit tests cover specific examples, edge cases, and error conditions. Property-based tests verify universal correctness across randomly generated inputs.

### Framework

- **Unit tests**: `pytest`
- **Property-based tests**: `pytest` + `hypothesis`
- Test files located in `tests/`
- Minimum **100 iterations** per property test (Hypothesis default is 100; set `@settings(max_examples=100)`)

### Property Test Configuration

Each property test is tagged with a comment referencing the design property:

```python
# Feature: face-attendance-arduino-display, Property N: <property_text>
@given(...)
@settings(max_examples=100)
def test_property_N_...(...)
```

### Unit Test Coverage

- `load_config` with present and absent `config.ini`
- `load_face_database` with valid CSV and missing CSV
- `CooldownTracker.should_send` at boundary (exactly 30s, just under, just over)
- `SerialSender` with mocked `serial.Serial`
- `Enroller` path construction
- `Recognizer` startup failure paths (mocked camera, mocked serial)

### Property Test Coverage

| Property | Hypothesis Strategy |
|---|---|
| P1: Face matching correctness | `st.lists(st.floats(-1,1), min_size=128, max_size=128)` for descriptors; `st.floats(0.01, 1.0)` for threshold |
| P2: Serial message format | `st.text(min_size=1, max_size=20, alphabet=st.characters(whitelist_categories=('Lu','Ll','Nd')))` |
| P3: Cooldown enforcement | `st.text(min_size=1)` for roll numbers; `st.floats(0, 29.99)` for elapsed time |
| P4: String truncation | `st.text(min_size=21)` for strings longer than 20 chars |
| P5: Config round-trip | `st.integers(0,9)`, `st.text()`, `st.integers(1200,115200)`, `st.floats(0.01,1.0)` |
| P6: Recognition log content | `st.text(min_size=1)` for roll number; `st.floats(0.0, 1.0)` for distance |
| P7: Enrollment path format | `st.text(min_size=1)` for roll number; `st.integers(1, 9999)` for index |
